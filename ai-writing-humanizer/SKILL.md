# AI Writing Humanizer

AIが書いた日本語文章を、読み手が「人が書いた」と感じる自然な文章に書き直す。

## 検出対象：AIっぽさの特徴

### 構造・表現の問題
- **テンプレ感**：前置き宣言（「結論から言うと」「本記事では」「以下で解説します」）
- **説明書感**：箇条書き多用、ラベル列挙（目的：背景：結論：）
- **過剰な丁寧さ**：締め文句（「参考になれば幸いです」「まずは小さく始めましょう」）
- **逃げ文句**：安全クッション（「一般的に」「多くの場合」「状況によって異なります」「一概には言えませんが」）
- **抽象語の空回り**：「重要」「効果的」「最適」「本質」「メリット」だけで押し切る
- **同義語連打**：「重要・大切・欠かせない」のような言い換えの繰り返し
- **抽象まとめ**：「まとめると」「要するに」「総じて」からの内容再掲

### 記号・表記の問題
- **Markdown記法**：`**太字**`、`##見出し`、箇条書き記号
- **「」の多用**：定義や強調のためのカギ括弧
- **()の多用**：補足を括弧に逃がす
- **コロン直後のスペース**：「目的： 〜」（禁止）
- **スラッシュ・矢印**：「A／B」「A→B」の並列表記

### 文章リズムの問題
- **均一な文長**：短い文と長い文の混在がない
- **型の連続**：断定→理由、結論→補足の繰り返し
- **接続詞過多**：必要以上の接続詞使用
- **人称の混乱**：私／筆者／私たちの混在

## 改善ワークフロー

1. **検出フェーズ**
   - 上記チェックリストに照らして問題箇所を特定
   - 問題の種類と箇所を整理

2. **改善フェーズ**
   - 意味と事実関係は変えない
   - 曖昧な箇所は曖昧なまま
   - 動詞中心の具体的表現に置き換え
   - Markdown記法をすべて除去
   - 括弧・記号を文脈に溶かす
   - 文のリズムを変化させる
   - 人称を一貫させる

3. **出力フェーズ**
   - 書き換え後の文章のみ出力
   - 解説・前置き・チェックリストは出さない
   - 元の段落構成を大きく崩さない
   - 文章量は元文から±20%以内

## 禁止事項

- 内容の捏造
- 根拠のない具体化（元文にない数字・固有名詞・事例を追加）
- 読者への質問や確認
- 前置き宣言の追加
- 締めの定型句の追加

## 出力形式

改善した文章のみを出力。「以下が改善後です」「〜を修正しました」などのコメントは不要。

## 使用例

### 入力例
```
**はじめに**
本記事では、効果的なプレゼンテーションについて解説します。一般的に、プレゼンテーションで重要なのは「内容」と「伝え方」です。
これらは非常に大切であり、欠かせない要素と言えるでしょう。
まとめると、以下のポイントを押さえることが本質的に重要です：
- 論理的な構成
- 視覚的なデザイン
参考になれば幸いです。
```

### 出力例
```
プレゼンテーションは内容と伝え方で決まる。何を言うかだけでなく、どう見せるか。論理的な構成で聞き手を迷わせず、視覚的なデザインで記憶に残す。この二つが揃って、はじめて相手に届く。
```
