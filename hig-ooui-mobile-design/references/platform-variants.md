# Platform Variants

## iOS (Baseline)
- Design for iPhone first unless instructed otherwise.
- Use standard iOS navigation patterns and system controls.

## iPadOS
- Consider split view and multi-column layouts.
- Support pointer interactions and keyboard shortcuts.
- Use sidebars for large information sets.

## watchOS
- Keep interactions glanceable and brief.
- Prefer single-purpose screens and short flows.
- Minimize text entry and use quick actions.

## Android (Delta Guidance)
- Map core objects and flows to Material patterns.
- Use system back behavior consistently.
- Use 48dp touch targets and Material typography.
